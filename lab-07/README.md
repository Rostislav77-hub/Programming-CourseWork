# Лабораторна робота №7: Реактивна взаємодія (EventBus / EventEmitter)

## Структура файлів
- `lib/eventBus.js` — кастомна реалізація патерну "Видавець-Підписник" (Event Bus), написана з нуля без використання вбудованого модуля `events` Node.js.
- `example/example7.js` — демо-кейс (симуляція роботи торгового майданчика із незалежними сутностями: Магазином, Складом, Логістикою тощо).

## Архітектура системи (Паттерн Publisher-Subscriber)
Система побудована на принципах слабкої зв'язності (loose coupling). Сутності не знають одна про одну безпосередньо, а спілкуються виключно через центральну шину повідомлень.
Система складається з наступних елементів:
1. **Центральне ядро (`EventBus`):** Керує підписками (`subscribe`, `subscribeOnce`), безпечною відпискою за ID (`unsubscribe`) та розсилкою подій (`publish`).
2. **Видавці подій (Publishers):** Генерують події в систему (наприклад, поява нового замовлення `order:new` або критичний залишок товару `stock:low`).
3. **Незалежні слухачі (Subscribers):** - *Склад* та *Логістика* паралельно реагують на нові замовлення.
   - *Аналітика* веде наскрізний підрахунок.
   - *Аудит* використовує одноразову підписку (`subscribeOnce`), реагуючи лише на першу подію.
   - Демонструється життєвий цикл: динамічна відписка Складу від оновлень без порушення роботи інших слухачів.
