My Labs (2-nd semester)

## 📂 Programming Labs

This repository contains completed labs. Each lab is formatted as an independent module with its own logic (`lib/`) and usage examples (`example/`).

### 📌 Completed labs:

- **[Lab 01 & 02] Генератори, Ітератори та Архітектура** ⚙️
  Реалізація нескінченних генераторів (Round Robin) та функції-споживача з таймаутом. В рамках цих завдань також була налаштована правильна архітектура проекту, ініціалізація Git, npm і поділ коду на модулі, що перевикористовуються.

- **[Lab 03] Просунута Мемоїзація (Memoization)** 🧠
  Функція вищого порядку для кешування результатів ресурсомістких обчислень. Реалізовано "розумні" стратегії очищення пам'яті при переповненні кешу: LRU (Least Recently Used), LFU, Time-Based та підтримка кастомних політик.

- **[Lab 04] Двостороння Черга з Пріоритетами (Priority Queue)** ⏳
  Гнучка структура даних для керування завданнями. Дозволяє витягувати елементи як за рівнем важливості (найвищий/нижчий пріоритет), так і за хронологією додавання (найстаріші/найновіші — FIFO/LIFO) з оптимізацією швидкості пошуку.

- **[Lab 05] Асинхронний Map (Async Array Functions)** ⚡
  Кастомна реалізація методу `array.map()` для роботи з асинхронними операціями. Включає дві версії: на класичних коллбеках (паралельна обробка) та на сучасних Promise/async-await з підтримкою примусового скасування процесу через `AbortController`.

- **[Lab 06] Потокова обробка великих даних (Async Iterators)** 🌊
  Система для інкрементальної обробки великих масивів даних. Використання асинхронних ітераторів (`for await...of`) дозволяє фільтрувати та трансформувати дані "на льоту" без переповнення пам'яті (OOM). Оновлена версія включає відмовостійку архітектуру конвеєра з явною передачею помилок (re-throw) та збереженням прогресу (partial stats) при аварійних збоях джерела.

- **[Lab 07] Реактивна взаємодія (EventEmitter / EventBus)** 📡
  Кастомна реалізація патерну "Видавець-Підписник" (Publisher-Subscriber) з нуля, без використання вбудованих модулів Node.js. Забезпечує слабку зв'язність між сутностями системи. Включає ізоляцію збоїв обробників (try/catch всередині emit), підтримку одноразових підписок (`subscribeOnce`) та виділений канал для безпечної маршрутизації помилок.

- **[Lab 08] Аутентифікаційний Проксі (Authentication Proxy & Strategy)** 🛡️
  Реалізація проксі-сервера для перехоплення та безпечної модифікації HTTP-запитів до API. Використано архітектурні патерни `Proxy` (через ланцюжок декораторів) та `Strategy` для динамічного перемикання між методами авторизації (API Key, JWT) "на льоту". Система включає автоматичне оновлення прострочених токенів (Token Renewal) при помилці 401, захист від спаму (Rate Limiting) та наскрізне логування процесів.
