# Лабораторна робота №6: Обробка великих даних (Async Iterators / Streams)

## Структура файлів
- `lib/dataStream.js` — реалізація логіки конвеєра: генератор-джерело, трансформер та споживач.
- `example/example.js` — демо-кейс із налаштуванням фільтрації, мапінгу та пакетної обробки (batching).

## Архітектура конвеєра (Pipeline)
Система побудована на базі асинхронних ітераторів, що дозволяє обробляти необмежені обсяги даних із мінімальним використанням оперативної пам'яті (Memory Efficiency).

1. **Джерело (`createDataStream`):** Асинхронний генератор, що імітує читання з БД. Використовує блок `try/finally` для гарантованого звільнення ресурсів навіть при аварійному завершенні.
2. **Трансформер (`processStream`):** Проміжна ланка, яка виконує фільтрацію та трансформацію об'єктів «на льоту». Кожна операція обгорнута у `try/catch` для ідентифікації ланки, на якій стався збій.
3. **Споживач (`consumeStream`):** Фінальна ланка, яка накопичує дані у батчі (batches) заданого розміру. Це дозволяє оптимізувати кількість записів у базу даних або зовнішні API.

## Обробка помилок (Error Propagation)
У цій версії реалізовано стратегію «явної передачі помилок»:
- **Re-throw:** Помилки в продюсері або трансформері не «з'їдаються», а прокидаються далі через `throw err`. Це запобігає тихому завершенню потоку (`done: true`) при виникненні збоїв.
- **Partial Stats:** При виникненні помилки споживач додає до об'єкта помилки поле `partialStats`. Це дозволяє системі знати, скільки саме даних було успішно оброблено до моменту аварії.
