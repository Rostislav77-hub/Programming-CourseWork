# Лабораторна робота №6: Обробка великих даних (Async Iterators)

## Структура файлів
- `lib/dataStream.js` — реалізація логіки генераторів та потоків.
- `example/example.js` — демо-кейс (обробка 1 000 000 записів із доведенням ефективності використання пам'яті).

## Архітектура конвеєра (Pipeline)
Система складається з трьох незалежних модулів, поєднаних через `for await...of`:
1. **Джерело (`createDataStream`):** Асинхронний генератор, що імітує читання записів з БД або файлу по одній одиниці.
2. **Трансформер (`processStream`):** Проміжний генератор, який фільтрує потік та змінює структуру даних "на льоту". Непотрібні записи одразу видаляються з пам'яті (Garbage Collection).
3. **Споживач (`consumeStream`):** Функція, що накопичує відфільтровані дані у невеликі пачки (batches) заданого розміру та передає їх на подальше збереження, після чого очищує локальний масив.

## Запуск 
Для перевірки роботи конвеєра та заміру оперативної пам'яті у термінал:
```bash
node example/example.js
